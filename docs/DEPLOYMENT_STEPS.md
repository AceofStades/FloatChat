# FloatChat AWS CLI Deployment Guide

This document contains the exact AWS CLI commands required to provision and deploy the FloatChat infrastructure across all 11 AWS services mentioned in the deployment architecture.

> **Note**: Replace the placeholders like `<ACCOUNT_ID>`, `<REGION>`, and `<ROLE_ARN>` with your actual deployment account values. In the original deployment:
> - **Account ID:** `121834500644`
> - **Region:** `ap-south-1`

---

## 1. Edge & Frontend Delivery (S3 + CloudFront)

### Amazon S3 (Static Website Hosting)
Create the frontend bucket, enable public access, and configure it for static hosting.

```bash
# Create the bucket
aws s3 mb s3://floatchat-frontend --region ap-south-1

# Disable Block Public Access (to allow public reads for the website)
aws s3api put-public-access-block \
    --bucket floatchat-frontend \
    --public-access-block-configuration "BlockPublicAcls=false,IgnorePublicAcls=false,BlockPublicPolicy=false,RestrictPublicBuckets=false"

# Attach a public read bucket policy
aws s3api put-bucket-policy \
    --bucket floatchat-frontend \
    --policy '{"Version":"2012-10-17","Statement":[{"Sid":"PublicReadGetObject","Effect":"Allow","Principal":"*","Action":["s3:GetObject"],"Resource":["arn:aws:s3:::floatchat-frontend/*"]}]}'

# Enable website hosting
aws s3 website s3://floatchat-frontend --index-document index.html --error-document 404.html

# Sync the compiled Next.js output to S3
aws s3 sync out/ s3://floatchat-frontend
```

### Amazon CloudFront (CDN)
Create a distribution pointing to the S3 bucket's website endpoint.

```bash
aws cloudfront create-distribution \
    --origin-domain-name floatchat-frontend.s3-website.ap-south-1.amazonaws.com \
    --default-root-object index.html
```

## 2. Security & Ingress Layer (WAF + API Gateway + Cognito)

### Amazon Cognito
Create the User Pool for managing identities.

```bash
aws cognito-idp create-user-pool \
    --pool-name FloatChatUsers \
    --auto-verified-attributes email \
    --policies '{"PasswordPolicy":{"MinimumLength":8,"RequireUppercase":true,"RequireLowercase":true,"RequireNumbers":true,"RequireSymbols":true}}'

# Create an app client for the Next.js frontend to interact with
aws cognito-idp create-user-pool-client \
    --user-pool-id <USER_POOL_ID> \
    --client-name FloatChatWebClient \
    --no-generate-secret
```

### AWS WAFv2
Create a Regional Web ACL to attach to API Gateway.

```bash
aws wafv2 create-web-acl \
    --name FloatChatWAF \
    --scope REGIONAL \
    --default-action Allow={} \
    --visibility-config SampledRequestsEnabled=true,CloudWatchMetricsEnabled=true,MetricName=FloatChatWAFMetrics \
    --region ap-south-1
```

### Amazon API Gateway
Create the REST API.

```bash
aws apigateway create-rest-api \
    --name "floatchat-rest-api" \
    --endpoint-configuration types=REGIONAL \
    --region ap-south-1

# (Further resources, methods, and integrations are created using `create-resource`, `put-method`, and `put-integration` for routes like /chatbot-response and /upload-data)
```

## 3. Serverless Compute Engine (ECR + Lambda + CloudWatch)

### Amazon ECR
Create the Elastic Container Registry to hold the backend Docker image.

```bash
aws ecr create-repository \
    --repository-name floatchat-backend \
    --region ap-south-1

# Authenticate Docker to ECR
aws ecr get-login-password --region ap-south-1 | docker login --username AWS --password-stdin 121834500644.dkr.ecr.ap-south-1.amazonaws.com

# Build and push the Docker image
docker build -t floatchat-backend ./server
docker tag floatchat-backend:latest 121834500644.dkr.ecr.ap-south-1.amazonaws.com/floatchat-backend:latest
docker push 121834500644.dkr.ecr.ap-south-1.amazonaws.com/floatchat-backend:latest
```

### AWS Lambda
Create the serverless function from the Docker image.

```bash
# Create the main API Lambda function with 1024MB Memory and a 5-minute timeout
aws lambda create-function \
    --function-name floatchat-api \
    --package-type Image \
    --code ImageUri=121834500644.dkr.ecr.ap-south-1.amazonaws.com/floatchat-backend:latest \
    --role <LAMBDA_EXECUTION_ROLE_ARN> \
    --memory-size 1024 \
    --timeout 300 \
    --region ap-south-1
```

### Amazon CloudWatch
Logs are automatically generated by Lambda, but you can explicitly create a log group.

```bash
aws logs create-log-group \
    --log-group-name /aws/lambda/floatchat-api \
    --region ap-south-1
```

## 4. Data, State, and Governance (DynamoDB + S3 + EC2 + CloudTrail)

### Amazon DynamoDB
Create the three required tables for application state.

```bash
# Users Table
aws dynamodb create-table \
    --table-name Users \
    --attribute-definitions AttributeName=userId,AttributeType=S \
    --key-schema AttributeName=userId,KeyType=HASH \
    --billing-mode PAY_PER_REQUEST \
    --region ap-south-1

# ChatHistory Table
aws dynamodb create-table \
    --table-name ChatHistory \
    --attribute-definitions AttributeName=userId,AttributeType=S AttributeName=timestamp,AttributeType=S \
    --key-schema AttributeName=userId,KeyType=HASH AttributeName=timestamp,KeyType=RANGE \
    --billing-mode PAY_PER_REQUEST \
    --region ap-south-1

# Sessions Table
aws dynamodb create-table \
    --table-name Sessions \
    --attribute-definitions AttributeName=sessionId,AttributeType=S \
    --key-schema AttributeName=sessionId,KeyType=HASH \
    --billing-mode PAY_PER_REQUEST \
    --region ap-south-1
```

### Amazon S3 (User Data Storage)
Create the backend bucket for processed `.nc` and `.db` files.

```bash
# Note: Bucket names must be globally unique
aws s3 mb s3://floatchat-user-data-121834500644 --region ap-south-1

# Secure the bucket by blocking public access
aws s3api put-public-access-block \
    --bucket floatchat-user-data-121834500644 \
    --public-access-block-configuration "BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true"
```

### Amazon EC2 (AI Proxy Server)
Launch the EC2 instance for the Ollama engine.

```bash
# Run a t3.micro instance with a specific key pair and tag
aws ec2 run-instances \
    --image-id <UBUNTU_AMI_ID> \
    --instance-type t3.micro \
    --key-name aws-key \
    --tag-specifications 'ResourceType=instance,Tags=[{Key=Name,Value=floatchat}]' \
    --security-group-ids <SECURITY_GROUP_ID> \
    --subnet-id <SUBNET_ID> \
    --region ap-south-1
```

### AWS CloudTrail
Set up the governance audit log.

```bash
# Create a dedicated S3 bucket for CloudTrail logs
aws s3 mb s3://floatchat-cloudtrail-logs-121834500644 --region ap-south-1

# Attach a bucket policy allowing CloudTrail to write to it
aws s3api put-bucket-policy \
    --bucket floatchat-cloudtrail-logs-121834500644 \
    --policy '{"Version":"2012-10-17","Statement":[{"Sid":"AWSCloudTrailAclCheck20150319","Effect":"Allow","Principal":{"Service":"cloudtrail.amazonaws.com"},"Action":"s3:GetBucketAcl","Resource":"arn:aws:s3:::floatchat-cloudtrail-logs-121834500644"},{"Sid":"AWSCloudTrailWrite20150319","Effect":"Allow","Principal":{"Service":"cloudtrail.amazonaws.com"},"Action":"s3:PutObject","Resource":"arn:aws:s3:::floatchat-cloudtrail-logs-121834500644/AWSLogs/121834500644/*","Condition":{"StringEquals":{"s3:x-amz-acl":"bucket-owner-full-control"}}}]}'

# Create and start the Trail
aws cloudtrail create-trail \
    --name FloatChatAuditTrail \
    --s3-bucket-name floatchat-cloudtrail-logs-121834500644 \
    --is-multi-region-trail \
    --region ap-south-1

aws cloudtrail start-logging \
    --name FloatChatAuditTrail \
    --region ap-south-1
```
